Processing Steps

Processing is controlled by an instance of IExtractionCoordinator. To verify that mandatory parameters are present, the ExtractionCoordinator can provide a list of missing prerequisites.

Invoking the ExtractDataAsync() method triggers the following steps:

  1. Check for prerequisite files, if any are missing report and abort.
  2. Create output folder containing timestamped name.
  3. Load data in parallel:
    • EIS snapshot data downloaded from EPA.
      • Reference value lookup tables for data validation:
        • SCC Codes
        • Control Measure Codes
        • NAICS Codes
        • Unit Type Codes
      • Downloaded facility snapshot XML data for determination of changed sources.
    • CAIS inventory data:
      • FacilitySite
      • ReleasePoint
      • EmissionsUnit
      • EmissionsProcess
      • ProcessPollutant
    • Optional Plant lists from a Excel workbooks for inclusion or exclusion in the submittal.
  4. Save Excel workbooks containing downloaded CAIS and EIS tables.
  5. Sequentially transform CAIS data to corresponding EIS properties and load Node facade tables:
    1. Facilities
    2. ReleasePoints
    3. Units
    4. Processes
    5. Controls
    6. Emissions
  6. Save Excel workbooks containing processed CAIS and Node tables.
  7. Save Node Facade tables as CERS XML files. This step may be skipped to expedite QA-only (dry run) extraction.
  8. Save an Excel workbook containing log and QA report.
