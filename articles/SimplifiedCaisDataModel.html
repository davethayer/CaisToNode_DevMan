Simplified CAIS Data Model

  • Plant in CAIS corresponds to EIS FacilitySite. Information at this level is not carried forward on a historical basis. Information at this level is changed to reflect updated data.
    • Data Year: As APEN updates are submitted, the most recent inventory information below the plant level is copied to a new data year, and this new year is edited to reflect the updated data For the purpose of EIS reporting, the data for each plant is taken from the latest data year in effect as of the EIS inventory year.
      • Stack in CAIS corresponds EIS ReleasePoint level.
      • Point in CAIS corresponds to EIS EmissionsUnit level.
        • Process in CAIS corresponds to EIS Process level. Each process has a single stack associated with it. Any stack can have multiple associated processes.
          • Process Pollutant in CAIS does not correspond to any specific EIS level per se, but it contains information represented at various levels in EIS.
            • Actual estimated emissions are reported in EIS ReportingPeriod level.
            • Potential emissions, emission limits, actual emissions, and hours of operation are used to estimate plant-wide potential emissions.
            • Control equipment and efficiency are used in EIS ControlMeasure and ControlPollutant levels.

AERR Source Type

A facility's AERR source type is used to determine reporting frequency. Type "A" large sources are required to report annually. This source type is determined based upon potential emissions and the source location's attainment or non-attainment status.

Control Path Implementation

CERS v2 specifies a very flexible model for reporting control equipment and release point allocation. Due to the basic nature of CAIS implementation only a dumbed-down subset of this scheme is reported.

  • In a CAIS process, all pollutant-level controls having the same code are considered to refer to the same piece of equipment.
  • Each process's controls are represented separately in the EIS FacilitySiteControl level. This is likely to result in multiple FacilitySiteControl records representing individual control devices.
